# docs

Game documentation, guides, and references for the team.

This folder is for anything that helps people understand or contribute to the project — not just programmers. Some ideas for what to put here:

- **Game design** — mechanics, progression, level design notes
- **Art direction** — style guides, colour palettes, reference sheets
- **Audio direction** — music mood boards, sound effect naming conventions
- **Writing** — narrative outlines, dialogue guidelines, tone references
- **Technical** — architecture decisions, system overviews, API notes
- **Onboarding** — setup instructions for new team members
- **Playtesting** — session plans, feedback templates, analysis notes

Use whatever format works for your team — Markdown, PDF, images, spreadsheets. The point is that knowledge lives in the repo, not in someone's head.
